Formatação de conteúdo HTML

Apesar de o conteúdo de uma página html poder ser escrito de qualquer maneira e com qualquer formatação para este funcionar, existem convenções seguidas por quase todos os programadores para facilitar a cooperação entre os mesmos.

Embora o ideal seja estar a par das convenções usadas para a empresa para a qual se trabalhe, esta página lista algumas das mais comuns para a morfologia de código HTML. As próximas página sirão focar-se na formatação dos elementos em geral ou de um elemenento específico.

Formatação geral

<br>

Quando se começa a aprender HTML e ainda não se conhece e/ou domina CSS, é bastante comum fazer uso de <br> para criar espaçamento entre conteúdo disposto na página.

Contudo, a função deste elemento é criar uma quebra de linha, para qualquer situação em que se tenha uma linha comprida que se queira dividir, sem ter de criar um novo elemento <p>, que tem a sua própria formatação.

Como tal, espaçamento e distanciamento de elementos dispostos na página devem ser feitos com reccurso a CSS em vez de <br>.

<hr>

O mesmo aplica-se a <hr> para criar divisórias. <hr> tem a semântica de "fim de uma secção e começo de outra", semelhante a <section> </section>, que surgiu em HTML5.

Como o elemento <section> já indica isto intrinsicamente, <hr>, este elemento é mais usado para criar quebras temáticas ou de tópicos.

Fazendo uma extensão a partir da informação anterior, pode-se dizer que <hr> é, em parte, um elemento semântico, assim como visual, que pode ser usado em vez de <div></div> para criar uma quebra visual, que pode também ser editada em CSS. Contudo, não é um susbtituto para margens, bordas e outras propriedades CSS.

Indentação

Para ajudar na leitura de código HTML, é extremamente aconselhável o uso de de indentação para ajudar a identificar os elementos criança e os elementos pai. Isto não só facilita a leitura, como também facilita a identificação de elementos e quais herdam propriedades.

Exemplo sem indentação 

<body>
<p>
Isto é um parágrafo
<ul>
<li>
Item 1 da lista
</li>
<li>
Item 2 da lista
</li>
</ul>
<p>
<body>

Exemplo sem indentação 

<body>
  <p>
    Isto é um parágrafo
    <ul>
      <li>
        Item 1 da lista
      </li>
      <li>
        Item 2 da lista
      </li>
    </ul>
  <p>
<body>

Como se pode ver nestes dois exemplos, o código é significaticamente mais fácil de ler quando está devidamente indentado.

Deve-se ainda ter em conta que há alguns elementos que não devem ser indentados, que podem ser referidos como crianças diretas. Os elementos em questão são <html>, <head> e <body>.

Comprimento de linha

Seja qual for o editor de texto ou IDE que se esteja a usar, no geral, estes não têm as dimensões do ecrã para a quebra de linha, de modo que é aconselhável fazer quebras manuais para manter o código e o texto facilmente legíveis.

A recomendação comummente encontrada é para limitar a 80 carateres. Pode-se fazer o argumento, indubitavelmente válido, de que com as dimensões de ecrã cada vez maiores, assim como a opção de moldar o texto, as linhas podem ser mais compridas. Na minha opinião pessoal, manter o limite de 80 carateress ajuda nos seguintes pontos:

  • Como a mancha textual é comparativamente mais pequena, pode ser mais fácil não perder a linha que se está a ler e também ser mais fácil encontrar algum segmento de texto.
  • É bastante comum, à medida que se vão adicionando elementos e até linguagens a uma página web, que hajam partes que tenham dezenas de carateres de indentação. Manter o limite de 80 caracteres, além de ajudar colaboradores que possam estar a usar ecrãs de menores dimensões, favorece também as pessoas que tenham ecrãs maiores quando eventualmente se tenham 8 a 10 elementos cirança para cima.

Existem algumas exceções a esta regra, contudo:

  • No elemento head, a informação para a tag <meta> tem de estar toda numa linha.
  • URLs não podem ser divididas. Se uma URL for demasiado grande, pode-se optar por a colocar noutra linha, junto com o atributo href, claro.
  • Embora esta seja uma guia geral que possa ser bastante útil, não vale a pena implementá-la ao ponto que o significado do código é alterado. Infelizmente, nem sempre dá para evitar linhas longas.

Escapamento de carateres (caracter escape)

Há símbolos que são usados para a marcação de texto HTMl, que se escritos normalmente, podem ser confundidos como parte da marcação e levar a resultados indesejados. Estes carateres são no geral <, >, &, " e '.

Para se fazer o escapamento destes carateres de modo a que sejam renderizados corretamente, deve-se fazer uso das entidades de carateres que se podem encontrar facilmente na web. Como referência rápida, as entidades destes carateres são &lt;, &gt;, &amp;, "quot; e &apos;.

Entidades

Ao escrever entidades de carateres em HTML, tendo em conta que normalmente se faz uso de UTF-8 para a codificação de carateres, estes podem ser escritos diretamente com o respetivo atalho de teclado (por exemplo, ©, € em vez de &copy;, &euro;). Nao só o texto que se inclui no código fica mais claro como se usam menos carateres. Em adição, só é precio ter em conta as entidades para os carateres de marcação html.

Comentários

O uso de comentários é essencial, seja para quando se trabalha em equipa e se querem colocar observações para guiar os outros membros do grupo, ajudar a dividir a estrutura do documento e/ou deixar lembretes para quando se retornar ao ficheiro em questão noutra altura.

Comentários de uma linha

Quando os comentários são apenas uma linha, tanto as tags como o comentário devem estar na mesma linha:

Comentário de uma linha errado 1 

<!--
Comentário
-->

Este comentário está errado porque <!--, Comentário e --> deviam estar na mesma linha.

Estes comentários, dentro das tags de abertura e fecho devem ser rodeados por espaços. Além de ajudar com a leitura dos comentários, isto deve-se ao facto de alguns caracteres não poderem ser usados diretamente a seguir ou antes das tags de comentário:

Comentário de uma linha errado 2 

<!--Comentário-->

Este comentário está errado porque não há espaçamento entre as tags do comentário e o comentário em si,

Comentário de uma linha correto 

<!-- Comentário -->

Comentários multilinha

As tags de comentários multilinha devem estar nas suas próprias linhas:

Mau exemplo de comentário multilinha 1 

<!-- Isto é um comentário multilinha
    Que se estende por várias linhas
-->

Este exemplo está errado porque a tag de abertura e o começo do comentário estão na mesma linha.

O comentário não deve estar indentado:

Mau exemplo de comentário multilinha 2 

<!--
  Isto é outro
	grande comentário
-->

Este exemplo está errado porque o comentário está indentado.

Bom exemplo de comentário multilinha 

<!--
Mais outro comentário
que se estende por várias linhas
-->

Informação sensível

Tendo em conta que qualquer pessoa pode inspecionar o elemento de página, e através daí, ter acesso ao código da página e os comentários da mesma, não se deve incluir informação sensível nestes.
Este tipo de informação pode ser usada por possíveis atacantes para mapear a estrutura e os ficheiros da aplicação, expôr partes ocultas do website como por exemplo o backend, entre outras ações maliciosas.

Exemplo de comentário com informação sensível 

<!--
O input dos campos deste formulário é enviado para 
a página processamento.html antes de ser transferida 
para a base de dados.
-->
