Tags, atributos e divs

As tags que compões os elementos HTML, assim como os atributos que os caraterizam, são a fundação para a escrita desta linguagem.

Para facilitar a compreenssão destes por membros de equipa ou aprendizados, devem-se tomar em conta algumas das convenções mais comuns, e outras opcionais. Segue-se assim a listagem e caraterização das mesmas.

Nesta página inclui-se também uma nota acerca de divs, uma vez que estass são um elemento particularmente importante na estruturação de quaqluer página web.

Tags e elementos

Capitalização

As tags dos elementos HTML devem ser escritas em letras minúsculas.

Mau exemplo de capitalização 

<Article>
  <P>
  </p>
</ARTICLE>

Este exemplo está errado por causa de alguns dos elementos terem letra capitalizadas.

Bom exemplo de capitalização 

<article>
  <p>
  <p>
<article>

Este examplo mostra a capitalização correta de tags e elementos HTML.

Espaçamento

Não devem haver espaços entre a abertura e fecho da tag e a identificação da tag.

Mau exemplo de espaçamento 

< article>
  < p >
  </p >
</article>

Este exemplo está errado devido aos espaços entre a abertura e fecho da tags <article> e <p>

Bom exemplo de espaçamento 

<article>
  <p>
  <p>
<article>

Este exemplo mostra o espaçamento correto de tags e elementos HTML.

Elementos vazios

Os elementos vazios podem ser escritos tanto com a terminação >, mais comum em HTML, como com a terminação />, obrigatória em XHTML.

Tendo em conta que HTML suporta ambas as opções, é essencialmente ao critério do desenvolvedor. Contudo, sugerem-se as seguintes considerações.

Uma vez que HTML suporta ambas as opções, mas XHTML requer o uso obrigatório da terminação />, o uso desta última poderá ser vantajosa para a hipótese de se fazer conversão para XHTML em algum ponto futuro.

O último ponto a ter em conta é manter a consistência, tanto para facilidade de cooperação entre diferentes colaboradores, como para uma melhor organização.

Mau exemplo de elementos vazios 

<p>
  Lorem ipsum dolor sit 
  amet, <br> consectetur 
  adipiscing elit, sed do <br />
  eiusmod tempor incididunt ut labore 
  et <br> dolore magna aliqua. Ut 
  enim ad minim veniam.
</p>

Este exemplo está errado porque não há consistência no fecho de elementos vazios.

Mau exemplo de elementos vazios 

<p>
  Lorem ipsum dolor sit 
  amet, <br /> consectetur  
  adipiscing elit, sed do <br />
  eiusmod tempor incididunt ut labore 
  et <br /> dolore magna aliqua. 
  Ut enim ad minim veniam.
</p>

Este exemplo mostra uma possibilidade de consistência para elementos fechados.

Elementos de fecho opcional

À semelhança dos elementos vazios, o fecho de elementos opcionais está ao critério do desenvolvedor web. Pode-se não fechar para pessoas que não tenham dificuldade com a ausência dessas tags.

Contudo, estas são são requeridas ao invés de opcionais em XHTML, portanto caso haja a possibilidade de o website vir a ser convertido para XHTML, será vantajoso manter as tags opcionais.

Atributos

Capitalização

Os atributos, assim como os seus valores, devem ser escritos em minúsculas.

Mau exemplo de capitalização 

<img src="iMg" class="Content" id="MAIN">

Este exemplo está errado por causa dos atributos terem letras capitalizadas.

Bom exemplo de capitalização 

<img src="img" class="content" id="main">

Este exemplo mostra a capitalização correta de atributos.

Nomeação de valores

Se analisarmos os nomes multipalavra de alguns atributos, podemos ver que a separação é feita através de traços.

Tendo em conta a indicação anterior, para indicar diferentes palavras compositórias de um atributo, não se poderiam usar capitalizações. Assim, quando a definir os valores de atributos como id, div e name, as opções restantes são espaços, traços (-) e underscores (_).

Não é aconselhável incluir espaços porque esses valores seriam inválidos para uso com CSS, uma vez que espaço não é um carater aí permitido.

O uso de underscores, embora não particularmente problemático, pode dificultar a leitura de quando o valor está sublinhado como ao ser uma âncora para hiperligação.

Assim, no geral, o mais aconselhável é usar traços (-) para separar palavras quando a escrever o valor de um atributo, por facilitar a leitura na situação supracitada e por ficar mais coerente com as atributos e valores já existentes.

Relativamente a traços e underscores, podem ser feitas exceções para convenções CSS, como por exemplo BEM.

Finalmente, e logicamente, quando a definir os nomes para estes atributos, estes devem ser tão semânticos quanto possível para facilitar a sua identificação. Outro pequeno aparte é o facto de ser relativamente importante evitar o uso de carateres únicos a uma língua (como por exemplo ç, à, etc) para evitar possíveis confusões.

Mau exemplo de nomeação 

<div class="mc 1">
  <h1 class="cabeçalho prin">
  </h1>
</div>

Este exemplo está errado porque a primeira classe tem um espaço, e a segunda tem um "ç". O facto de as classes terem identificações tão crípticas pode também ser problemático.

Bom exemplo de nomeação 

<div class="main-content-1">
  <h1 class="main-header">
  </h1>
</div>

Este exemplo mostra uma possibilidade para nomeação correta de atributos. Nomes mais curtos, como no mau exemplo, podem ser aceitáveis havendo documentação apropriada.

Aspas e pelicas

Quando a escrever o valor de um atributo, este deve estar dentro de aspas. Embora as pelicas sejam uma opção, estas costumam ser mais usadas em linguagens de programação e gestão de base de dados, de modo que manter a consistência com aspas em HTML ajuda a separar as linguagens quando estas têm de ser conjugadas.

Mau exemplo de aspas e pelicas 

<div class= test>
  <div id='area-1'>
  </div>
  < div id="area-2">
  </div>
</div>

Este exemplo está errado porque a classe test não será devidamente identificada por não ter começo e fim definido (com aspas), e o id area-1 está delineado com pelicas em vez de aspas.

Bom exemplo de aspas e pelicas 

<div class="test">
  <div id="area-1">
  </div>
  < div id="area-2">
  </div>
</div>

Este exemplo mostra como o mau exemplo deveria estar aspado.

Atributos boleanos

Embora atributos booleanos possam ser escritos por completo, não é preciso para os definir, sendo a forma completa redundante.

Mau exemplo de atributos booleanos 

<input type="checkbox" checked="checked" 
name="cheese" disabled="disabled">

Este exemplo está errado porque os atributos required e disabled podem ser substituídos pelos seus valores.

Bom exemplo de atributos booleanos 

<input type="checkbox" checked 
name="cheese" disabled>

Este exemplo demonstra a sumarização correta deste elemento.

Ordem de atributos

Para facilitar a leitura de atributos, é importante seguir uma ordem dos mesmos. Por exemplo, como as classes são facilmente reusadas dentro do mesmo documento, é aconselhável estas surgirem antes dos id. Assim, de primeiro para último quando a ler a partir da esquerda:

  • src, for, type, href, value;
  • class;
  • id, name;
  • title, alt;
  • role, aria*;
  • data-*.

Divitis e classitis

Divitis consiste no uso excessivo de divs e classitis no uso excessivo de classes para "embrulhar" conteúdo para outros propósitos que não a divisão do conteúdo da página.

Embora isto resulte, pode resultar em código demasiado grande, lento e/ou consumidor de outros recursos que não seriam necessários.

Tendo em conta que o foco de HTML é ser uma linguagem de marcação estrutural, devemos fazer por o limitar o mais possível ao mínimo de elementos necessários ao colocar vários estilos num único elemento onde possível e usar os elementos estruturais e semânticos de html.

Exemplo de divitis e classitis 

<div class="right-arrow"></div>
<div class="artigo">
  <div class="lista de frutas">
    <ul>  
      <li>
        Maçã
      </li>
      <li>
        Banana
      </li>
      <li>
        Laranja
      </li>
    >/ul>
  </div>
</div>

Este exemplo está errado porque "right-arrow" pode ser dividido em arrow para ser reusado e right ser a classe que dá a rotação apropriada.

A div artigo pode ser substituída pelo elemento semântico article e a class "lista de frutas" pode ser colocada na ul, eliminando um div denecessária.

Correção do exemplo anterior 

<div class="right arrow"></div>
<article>
  <ul class="lista de frutas">  
    <li>
      Maçã
    </li>
    <li>
      Banana
    </li>
    <li>
      Laranja
    </li>
  >/ul>
</article>

Pode-se ver na div deste exempo que se podem aplicar várias classes a um elemento ao separá-las com um espaço. conseguem-se assim misturar classes gerais com classes específicas, promovendo a reciclagem das primeiras.

Observa-se também o uso do elemento semântico article para melhor clareza, assim também como a incluisão da class "lista de frutas" na ul, podendo assim omitir uma div extra.
